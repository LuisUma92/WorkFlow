# Lessons Learned

| Date       | What went wrong / What was learned | Rule to follow next time |
|------------|-----------------------------------|--------------------------|
| 2026-03-25 | First ADR draft stored exercise content (stem, solution) in DB. User corrected: .tex files are the truth source, DB is metadata index only. | Never propose storing authored content in DB when files are the canonical source. DB indexes metadata, files hold content. |
| 2026-03-25 | First ADR draft assumed MathJax would handle custom macros on institutional Moodle. User flagged: no control over institutional MathJax config. | Never assume external system configuration. Custom macros must be expanded to standard form before leaving the local system. |
| 2026-03-25 | Overlooked that LaTeX brace-parsing is needed by both tikz and exercise modules. User suggested abstracting into shared module. | When two modules need the same parsing logic, extract to a shared utility module immediately — don't duplicate. |
| 2026-03-25 | ADR-0010 didn't account for exercise file lifecycle (placeholder files from crete). User clarified the creation→completion flow. | Always model the full lifecycle of artifacts, not just their final state. Track status transitions explicitly. |
| 2026-03-25 | `MacroMatch.args` used mutable `list[str]` in frozen dataclass — caught by python-reviewer. | Always use `tuple` for collection fields in frozen dataclasses, never `list`. |
| 2026-03-25 | `upsert` guard `if value is not None` debated across reviewers. Python reviewer self-corrected: `0 is not None` → True, code was correct. Code reviewer was wrong. | When reviewers disagree, verify with actual Python semantics before acting. `is not None` correctly passes falsy values (0, False, ""). |
| 2026-03-25 | Tags filter used `Exercise.tags.contains(tag)` which is SQL LIKE substring match on JSON text — "math" matches "biomathematics". Three reviewers flagged. | JSON stored as text requires exact matching with delimiters (e.g., `'"{tag}"'`). Prefer proper JSON columns or M2M tables. |
| 2026-03-25 | `options` list built in sync but never attached to exercise record — silent data loss. Caught by both python and code reviewers. | After building child objects, always attach them to the parent before persisting. Dead code that constructs but doesn't save is a data loss bug. |
| 2026-03-25 | Detached ORM objects used after session close in build-exam CLI. Both python and code reviewers flagged. | Keep SQLAlchemy session open through the entire operation that uses ORM objects. Don't close session then iterate results. |
| 2026-03-25 | Path traversal in generator.py: exercise_id used directly in filename without sanitization. Security reviewer flagged. | Always validate user-provided strings used in filenames. Use allowlist regex (`^[a-zA-Z0-9._-]+$`). |
| 2026-03-25 | 4-reviewer parallel schema (python + code + security + TDD) consistently finds issues that single-pass review misses. Security finds path traversal, TDD finds untested branches, code reviewer finds architectural bugs. | Use the 4-reviewer schema for every non-trivial implementation phase. Saved to memory as feedback_review_schema.md. |
| 2026-03-25 | Parallel coding agents (Stream A + Stream B) built against the same domain types but used different attribute names (node_id vs id, source_id vs source). The CLI created a 60-line adapter shim to paper over it. 4 reviewers unanimously flagged CRITICAL. | When parallel agents share a domain type, create domain.py FIRST as a standalone task, verify it's on disk, then launch parallel agents that import from it. Never let agents define their own stubs of shared types. |
| 2026-03-25 | Export test files used monkeypatch to inject fake domain stubs, which masked the CRITICAL API mismatch. Tests passed but real usage would crash with AttributeError. | Never monkey-patch domain types in tests. Import the real types. If the real module doesn't exist yet, the test should FAIL, not pass with fakes. |
| 2026-03-25 | Note splitter accepted `%>` marker paths without validation. Security reviewer found CRITICAL path traversal — `%>../../.bashrc` would write outside output_dir. Same pattern as exercise generator.py (Phase 4d). | Any function that constructs file paths from user/file content must validate containment: `resolved.startswith(str(output_dir.resolve()))`. This is the THIRD time path traversal was caught — make it a reflex. |
| 2026-03-26 | `main.py` imported from nonexistent `app` module (`from app.iofunc`), blocking the entire CLI from loading. None of the 18 commands worked. Discovered only during smoke testing. | Always smoke-test CLI entry points (`workflow --help`) after wiring new commands. Unit tests don't catch import-time crashes in the entry point. |
| 2026-03-26 | Legacy `start` command in main.py used `eval()` on user input, undefined `cfg`/`options` variables, and broken imports. All pre-existing. | When cleaning up legacy code, remove broken commands entirely rather than trying to fix them. Dead code with `eval()` is a security liability. |
| 2026-03-26 | CI workflow linted `.` (everything) instead of `src/ tests/`, hitting flake8 internals in `.venv/` and installed packages. Also didn't `pip install -e .` to get project deps. | CI should lint `src/ tests/` specifically, install the project with `pip install -e .`, and test only maintained test directories. |
| 2026-03-26 | `build-eval` CLI command passed `pool=[]` (empty list) to `select_exercises` — effectively a no-op. Two reviewers flagged HIGH. The command looked functional but never actually selected exercises. | After implementing a CLI command, test it end-to-end with real data, not just with unit test mocks. Commands that silently produce empty output are worse than errors. |
| 2026-03-26 | User prefers model routing: architect=opus, TDD=sonnet, reviewers=opus, git-ops=haiku. This balances quality vs speed effectively. | Use this model routing pattern when user requests it: opus for planning/review (deep reasoning), sonnet for implementation (fast coding), haiku for git operations (lightweight). |

# Request: CLI para gestión de evaluaciones en ITeP/WorkFlow

## Contexto

La base de datos `workflow.db` tiene un esquema de 4 capas (ADR ITEP-0002) que
incluye `evaluation_template`, `item` y `evaluation_item`, pero **no existe un
comando CLI** para crear, listar o modificar estos registros. Hoy la única forma
de poblarlos es con scripts ad-hoc o directamente por SQLAlchemy.

## Comandos solicitados

### 1. `workflow eval list`

Listar templates de evaluación, opcionalmente filtrados por institución.

```
workflow eval list                    # todos
workflow eval list --inst UFide       # solo UFide
workflow eval list --inst UCR --full  # con detalle de ítems
```

Salida esperada (modo `--full`):

```
[UFide] Estudio de caso (63 pts, 6 ítems)
  1. SU - Información / Comprender      2 × 5 pts
  2. RC - Información / Análisis         1 × 5 pts
  3. Desarrollo - Proc. Mental / Aplicar 2 × 16 pts
  4. Desarrollo - Proc. Mental / Evaluar 1 × 16 pts
```

### 2. `workflow eval add`

Crear un nuevo template de evaluación interactivamente o por flags.

```
workflow eval add --inst UFide --name "Prueba parcial"
```

Flujo interactivo:
1. Seleccionar institución (si no se pasa `--inst`)
2. Nombre del template
3. Agregar ítems en loop:
   - Tipo (SU/RC/Desarrollo)
   - Dominio taxonómico (selector)
   - Nivel taxonómico (selector)
   - Cantidad y puntos por ítem
4. Confirmar y guardar

### 3. `workflow eval edit <id>`

Editar un template existente: agregar/quitar ítems, modificar puntos.

### 4. `workflow item list`

Listar ítems taxonómicos registrados, con filtros por dominio/nivel.

```
workflow item list
workflow item list --domain "Información"
workflow item list --level "Comprender"
```

### 5. `workflow item add`

Crear un ítem taxonómico nuevo.

```
workflow item add --name "SU - Info/Recordar" \
  --level Recordar --domain Información
```

### 6. `workflow course list`

Listar cursos registrados con su institución.

```
workflow course list
workflow course list --inst UCR
```

### 7. `workflow course add`

Crear un curso nuevo.

```
workflow course add --inst UFide --code FI-201 \
  --name "Física II" --lpw 3 --hpl 2
```

## Integración con Neovim

### Objetivo

Poder consultar y actualizar la DB sin salir de nvim, usando el plugin
`workflow.nvim` que ya existe (Phase 1 merged).

### Propuesta de comandos nvim

| Comando vim          | Acción                                              |
|----------------------|-----------------------------------------------------|
| `:WorkflowEvalList`  | Abre split con lista de templates (formato tabla)   |
| `:WorkflowEvalShow`  | Detalle del template bajo cursor o por argumento    |
| `:WorkflowItemList`  | Lista ítems taxonómicos en floating window          |
| `:WorkflowCourseList`| Lista cursos en floating window                     |

### Arquitectura sugerida

1. **CLI como backend**: Los comandos vim invocan `workflow eval list --json`
   (agregar flag `--json` a todos los subcomandos) y parsean la salida.
2. **Server LSP existente**: Alternativamente, exponer estos queries como
   métodos del server LSP que `workflow.nvim` ya levanta, usando el protocolo
   JSON-RPC existente. Esto evita spawning de procesos y permite caching.
3. **Telescope picker**: Para selección interactiva (ej. vincular un template
   a un curso), usar `telescope.nvim` con un finder custom que consulte la DB.

### Prioridad de implementación

1. **P0**: `workflow eval list` y `workflow item list` (solo lectura, útil hoy)
2. **P1**: `workflow eval add` y `workflow item add` (creación interactiva)
3. **P1**: Flag `--json` en todos los subcomandos (habilita integración nvim)
4. **P2**: Comandos nvim `:WorkflowEvalList`, `:WorkflowItemList`
5. **P2**: `workflow eval edit` y `workflow course add/list`
6. **P3**: Telescope pickers y métodos LSP server

## Fuentes de mejora adicionales

### En `itep/manager.py`

- **Falta**: funciones `list_evaluation_templates()`, `list_items()`,
  `list_courses()` con filtros. Solo existe `list_all()` genérico.
- **Falta**: `get_evaluation_template_detail()` que haga join con
  `evaluation_item` e `item` para mostrar el desglose completo.
- **Falta**: validación de duplicados al crear templates (mismo nombre +
  institución).

### En el esquema DB

- **Mejora**: campo `item_type` en `item` (SU, RC, Desarrollo) para filtrar
  sin parsear el nombre. Hoy el tipo está implícito en el `name`.
- **Mejora**: campo `total_points` calculado o view en `evaluation_template`
  (sum de `total_amount × points_per_item`).
- **Mejora**: `evaluation_template.description` para almacenar reglas de
  redacción (como las del estudio de caso UFide).

### En `workflow.nvim`

- **Phase 2 scope**: Los comandos `:Workflow*List` encajan en Phase 2 del
  plugin (que ya planea health check y telescope picker).
- **Quick win**: Statusline ya muestra proyecto; agregar conteo de
  evaluaciones registradas para el curso activo.
